# Review rubric

Use this compact rubric when reviewing Jadu work:

## 1. Scope

- Is the artifact actually the one the task asked for?
- Did execution stay inside the approved scope?

## 2. Quality

- Is the result complete enough to hand off?
- Are there visible breakages, omissions, weak transitions, or confusing sections?

## 3. Records

- Did the executor update the real Jadu files (`TASKS.md`, `WORKLOG.md`, and related context) when appropriate?
- Are findings turned into actionable unchecked tasks before more work continues?

## 4. Verification

- Was the result checked in a way that matches the artifact type?
- Are claims about completion supported by the artifact or verification output?

## Finding format

When you report a finding, include:

1. What is wrong
2. Where it is
3. Why it matters
4. What a passing fix must achieve

Do not approve while material unchecked findings remain.
